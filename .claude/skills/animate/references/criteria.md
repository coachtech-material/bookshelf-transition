## 1. 中心概念の絞り方

動画は Section の **要約ではなく予告編**。学習の最初に視聴し「この Section で何が起きるか」の地図を渡す。

- Section の 🎯（学ぶこと）から **主役の概念 1〜2 個** を選ぶ。本文の全トピックを網羅しない（細部は本文が担う）
- 🧠（メンター人格の語り）の痛み・比喩と呼応させると、本文との往復がスムーズになる
- まとめシーンは Section の ✨ まとめの 4 点以内に対応させる

## 2. 尺とシーン数

- **1.5〜3 分 / 5〜8 シーン**。ナレーション総量 700〜1,000 字（日本語は約 330 字/分）
- 1 シーンのナレーションは 60〜200 字（20〜36 秒）。これを超えるならシーンを割る

## 3. シーン型の選び方

| 伝えたいこと | シーン型 | 例 |
|---|---|---|
| Section の開始・主題 | `title` | 全 Section の 1 シーン目（固定） |
| 2 つの書き方・状態の対比、エラー体験 | `codeCompare` | 移行前 vs 移行後、誤りと正解 |
| 二項対立・分類の整理 | `keypoint` | 手段 A vs 手段 B の使い分け |
| 直感的なメンタルモデル | `figure`（既存概念図を再利用） | /illustrate の生成画像 |
| 処理・データ・手順の流れ | `flow` | 入力 → 変換 → 出力 → 分岐先 |
| 包含関係 | `nest` | 全体 ⊃ 部分 ⊃ 中核 |
| まとめ + 次への接続 | `outro` | 全 Section の最終シーン（固定） |

- `title` と `outro` は必須。間に 3〜6 シーンを構成する
- 同じ型を連続させてよい（codeCompare 2 連続など）。ただし 3 連続を超えるなら構成を見直す
- 新しい見せ方が必要なら、シーン型を `video/src/scenes/` に追加してから使う（公式 Skill のルールに従い `useCurrentFrame()` 駆動で実装）

## 4. 挿入位置と画像との共存

- 動画: **Section タイトル直後**（📝 前提知識の行・グループ全体像があればその後、🎯 の前）
- 概念図（/illustrate）: 従来どおり 🧠 直後。**動画は概念図を置き換えない**（figure シーンの素材として再利用する関係）

## 5. 冪等判定

1. Section ファイルのタイトル直後に `releases/download/videos/<sectionId>.mp4` を含む `<video>` タグ → **挿入済み**（スキップ）
2. タグが無く `video/out/<sectionId>.mp4` が存在 → **生成済み・未挿入**（配信と挿入のみ行う）
3. どちらも無い → 未生成（フル実行）
