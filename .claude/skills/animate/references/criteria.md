## 1. 動画の役割 — Section の主たる学習路

動画は Section の **要約でも予告編でもなく、その Section を理解するための主たる学習路** です。学習者は最初に動画を見て Section の内容を理解し、テキストは補助（細部の確認・コピペ・読み返し）として使う、というスタンスで設計します。

したがって動画は、Section の **核となる学びを最後まで成立させる** こと。「何が起きるかの地図だけ渡して本文に丸投げ」はしません。

- Section の 🎯（学ぶこと）と ✨（まとめ）を**すべてカバー**する。中心概念を 1〜2 個に絞って残りを本文へ流す、という作り方はしない
- 解説は **Why（なぜ必要か）→ What（何か）→ How（どう使うか）** の順で組む。導入は本文の「導入（課題・痛み）」に対応させ、🧠（メンター人格の語り）の比喩・実体験と呼応させる
- 各概念は「動画を見ただけで腑に落ちる」レベルまで噛み砕く。定義 → 最小コード例 → 「だから Laravel ではこう書く」の接続まで見せる
- ただし **本文に委ねてよいもの** はある: コピペ用の長いコード全文、細かい構文の網羅、外部リンク、コラム（💡/📝/⚠️）の枝葉。動画は「理解の幹」を通し、テキストが「枝葉と再現性」を担う分担にする

## 2. 尺とシーン数

- **3〜6 分 / 8〜14 シーン**（要約ではなく解説なので、従来の予告編より長い）。ナレーション総量 1,000〜1,800 字（日本語は約 330 字/分）
- 1 シーンのナレーションは 80〜200 字（約 16〜36 秒）。これを超えるならシーンを割る
- Section 本文が短い・概念が少ない場合は下限（3 分 / 8 シーン）に寄せてよい。字数を埋めるための水増しはしない（writing.md と同じ方針）

## 3. シーン型の選び方

| 伝えたいこと | シーン型 | 例 |
|---|---|---|
| Section の開始・主題・導入の問い | `title` | 全 Section の 1 シーン目（固定） |
| 2 つの書き方・状態の対比、エラー体験、コードの読み解き | `codeCompare` | 手続き的 vs OOP、移行前後、誤りと正解 |
| 二項対立・分類・定義の整理 | `keypoint` | 認証 vs 認可、プロパティ vs メソッド |
| 直感的なメンタルモデル（既存概念図の再利用） | `figure` | /illustrate の生成画像（🧠 の比喩を絵にしたもの） |
| 処理・データ・手順・ライフサイクルの流れ | `flow` | リクエスト → 認可 → 処理 → レスポンス |
| 包含関係・階層 | `nest` | 名前空間 ⊃ クラス、`v1` ⊃ コントローラ |
| まとめ + 次への接続 | `outro` | 全 Section の最終シーン（固定） |

- `title` と `outro` は必須。間に 6〜12 シーンを構成し、🎯 と ✨ を取りこぼさない
- 同じ型を連続させてよい（codeCompare 2 連続など）。ただし単調にならないよう、解説の山場では型を切り替える
- **codeCompare のラベル色** `labelColor`: 中立は省略（既定で青 `theme.primary`）。対比で色を分けたいときは左右で `theme.primary`（青 `#3B82F6`）と `theme.accent`（オレンジ `#F97316`）を割り当てる。PHP/Laravel のロゴ色などは使わず、教材全体のアクセント（青・オレンジ）に統一する
- 新しい見せ方が必要なら、シーン型を `video/src/scenes/` に追加してから使う（公式 Skill `remotion-best-practices` のルールに従い `useCurrentFrame()` 駆動で実装し、色は必ず `src/theme.ts` のトークンを参照する）

## 4. デザインの前提（全シーン共通）

- 配色は `src/theme.ts` のトークンに集約。**ライト・マット基調**（オフホワイトの面）に、青 `#3B82F6` とオレンジ `#F97316` をアクセントとして使う（GitHub Pages サイトのモノクロ基調・/illustrate の概念図と統一）。質感は「立体感・リッチ・マット・大人」
- シーンを実装・調整するときは色を直書きせず、必ず `theme.*` を参照する（基調を一括で変えられるようにするため）

## 5. 挿入位置と画像との共存

- 動画: **Section タイトル直後**（📝 前提知識の行・グループ全体像があればその後、🎯 の前）。学習者が最初に視聴する主経路
- 概念図（/illustrate）: 従来どおり 🧠 直後。**動画は概念図を置き換えない**（figure シーンの素材として再利用する関係。白背景の図はライト基調の上で「額装した紙」として馴染む）

## 6. 冪等判定

1. Section ファイルのタイトル直後に `releases/download/videos/<sectionId>.mp4` を含む `<video>` タグ → **挿入済み**（スキップ）
2. タグが無く `video/out/<sectionId>.mp4` が存在 → **生成済み・未挿入**（配信と挿入のみ行う）
3. どちらも無い → 未生成（フル実行）
