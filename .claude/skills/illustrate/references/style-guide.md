# プロンプトスタイルガイド

教材の概念図を Gemini で生成する際のプロンプト構成ルール。

## 基本ルール

| 項目 | ルール |
|------|--------|
| 言語 | **ラベル・キーワードは日本語** で指定する。英語の技術用語（API, REST 等）はそのまま使用可 |
| スタイル | フラットデザイン・インフォグラフィック風。影・グラデーション・写実表現は避ける |
| 背景 | 白背景（#FFFFFF） |
| 配色 | 青系（#3B82F6）とオレンジ系（#F97316）をアクセントカラーとする。対比図では左右で色を分ける |
| 線画 | アウトライン付きのクリーンな線画。手書き風にしない |
| 情報量 | 1枚に盛り込む概念は1つ。要素は5〜8個以内に抑える |
| 文字 | 大きく読みやすいラベルのみ。長文は入れない |

> ⚠️ **文字アーティファクトに注意**: Gemini は日本語ラベルを概ね正しく描くが、ラベルが多い・指示が曖昧だと、無関係な英単語（例: "CLICK"）や崩れた文字が混入することがある。生成後は必ず目視で確認し、混入したらラベルを減らす等で再生成する。

## プロンプト構成テンプレート

```
[画像の種類]を作成してください。

【内容】
- [何を描くか。比喩を使う場合はその対応関係を具体的に説明する]

【スタイル】
- フラットデザイン・インフォグラフィック風
- 白背景、青（#3B82F6）とオレンジ（#F97316）のアクセントカラー
- クリーンな線画、ラベルは日本語
- 16:9 横長レイアウト
```

**ポイント**: 【内容】セクションは具体的に書く。「○○の図」と曖昧に書くのではなく、「中央に△△、左側に□□、右側に◇◇、それらを矢印でつなぐ」のように、構図を明確に指定する。

## 画像の種類別ガイド

criteria.md の 3 つの構図に対応する。排他的ではなく、1 枚が複数を兼ねてよい。

### 比喩図（メンタルモデル）

抽象概念を日常の物・場面（工場・倉庫・建物・道具等）に置き換え、「どういう感覚で捉えればいいか」を伝える。🧠（[人格名]の語り）の比喩を絵にするのに最適。

- 中心に主概念を配置し、周囲に関連要素を配置する
- 技術用語と日常の比喩を **併記** する（例: 「倉庫番 = ○○コンテナ」）
- 対応関係が一目でわかる構図にする

### 対比図（Before/After）

技術の「なぜ必要か」や思考の転換を、左右の対比で視覚的に伝える。

- 左右に分割し、左に旧/課題、右に新/解決を配置する
- 矢印で転換を示す
- 左右で配色を分ける（左: グレー寄り、右: 青/オレンジのアクセント）

### 鳥瞰図（地図）

複雑なシステムの全体像を「地図」として 1 枚に収める。Mermaid の箱図では伝わりにくい「全体の形・配置の感覚」を担う。

- アイソメトリック（斜め上から見下ろす）視点で、街・施設の地図のように描く
- 全体の「形」を把握できる程度に簡略化する
- **各要素を日常の物に見立て、技術名と比喩ラベルを併記する**（例: DB = 地下金庫のある銀行、外部サービス = 取引先の建物、ロードバランサ = 交通整理員）
