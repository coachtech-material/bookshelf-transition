# 3-1-pivot-ledger

**モデル**: pro | **アスペクト比**: 16:9 | **解像度**: 2k

## プロンプト本文

```
多対多のピボットテーブルを「1行＝1組み合わせ」として、具体例で分かりやすく表す立体感のある概念図を作成してください。

【内容】
- 左側に立体的なタスクカードを2枚、縦に並べる
  - 上: 「タスクA」のカード。すぐ横にオレンジのタグチップ2つ（「仕事」「緊急」）
  - 下: 「タスクB」のカード。すぐ横にオレンジのタグチップ1つ（「仕事」）
- 左から右へ太い矢印を1本引き、近くに「組み合わせを1行ずつ記録」と添える
- 右側に「ピボットテーブル task_tag」とタイトルを書き、その下に横長の行バーを縦に3本だけ重ねる。各バーは1つの組み合わせを表し、左半分が青でタスク名、右半分がオレンジでタグ名、間に小さな「＋」。3本は上から順に:
  - 「タスクA ＋ 仕事」
  - 「タスクA ＋ 緊急」
  - 「タスクB ＋ 仕事」
- 上の2本（どちらもタスクA）を左脇の波括弧でまとめ、「1つのタスクに2つのタグ → 2行」と短く添える

【スタイル】
- フラットなインフォグラフィック／ベクター風を基調に、アイソメトリック（斜め上視点）で軽い立体感を出す（カードとバーに厚みを持たせる）。文字は読みやすく保つ
- 影はオブジェクト真下の単色フラット影のみ。ぼかし影・グラデーションは使わない
- 背景は #FFFFFF の単色。テクスチャ・しみ・もや・かすれを描かない
- タスクは青（#3B82F6）、タグはオレンジ（#F97316）。クリーンな線画、ラベルは日本語
- 行バーはちょうど3本にする。タイトルや無関係な英単語など、指定外の文字は入れない
- 16:9 横長レイアウト
```
