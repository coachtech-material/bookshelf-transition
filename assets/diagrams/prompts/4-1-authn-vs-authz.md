# 4-1-authn-vs-authz

**モデル**: pro | **アスペクト比**: 16:9 | **解像度**: 2k

## プロンプト本文

```
認証と認可の違いを「建物の入口ゲート」と「各自の鍵付きロッカー」の2段の比喩で表す、立体感のある概念図を作成してください。

【内容】
- 左側に立体的な建物の入口ゲートを描き、人物が ID カード（バッジ）をかざして通過している。ゲートの上に「認証: ログイン済みか」とラベル
- 右側に、通過した先の立体的なロッカー（個別の箱）を2つ並べる。1つは鍵が開いて中身（タスク）が見え、本人が操作できる様子。「自分のタスク 許可」とラベル
- もう1つのロッカーは施錠され、赤い禁止マークと「403」。「他人のタスク 不可」とラベル。右側全体の上に「認可: 所有者か」とラベル
- 左から右へ向かう矢印（ゲートを通った後に認可がある流れ）

【スタイル】
- フラットなインフォグラフィック／ベクター風を基調に、アイソメトリック（斜め上視点）で立体感を出す（ゲートやロッカーに厚みを持たせる）
- 影はオブジェクト真下の単色フラット影のみ。ぼかし影・グラデーションは使わない
- 背景は #FFFFFF の単色。テクスチャ・しみ・もや・かすれを描かない
- 認証側は青（#3B82F6）、認可側はオレンジ（#F97316）のアクセントカラー、禁止マークは赤。クリーンな線画、ラベルは日本語
- タイトルや無関係な英単語など、指定外の文字は入れない
- 16:9 横長レイアウト
```
