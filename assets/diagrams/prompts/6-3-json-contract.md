# 6-3-json-contract

## プロンプト本文

```
API テストが「約束した JSON の型」に合うかを固定する様子を、立体感のある概念図で表してください。

【内容】
- 中央に、決まった形のくぼみを持つ立体的な「型（金型・テンプレート）」を1つ置く。型のくぼみは2段に分かれ、それぞれ「data」「meta」とラベル。型の下に「約束した JSON の型」とラベル（この見出しは1か所だけ）
- 左から、その型のくぼみにぴったりはまる正しいデータブロック（「data」「meta」の区画を持つ）が収まり、大きな緑のチェック。「形が合う → 合格」とラベル
- 右に、形が崩れて型に合わないデータブロック（区画が欠けている）が、大きな赤い禁止マークではじかれている。「形崩れ → 赤で停止」とラベル

【スタイル】
- フラットなインフォグラフィック／ベクター風を基調に、アイソメトリック（斜め上視点）で立体感を出す（型・ブロックに厚みを持たせる）
- 影はオブジェクト真下の単色フラット影のみ。ぼかし影・グラデーションは使わない
- 背景は #FFFFFF の単色。テクスチャ・しみ・もや・かすれを描かない
- 青（#3B82F6）とオレンジ（#F97316）のアクセント、合格は緑・失敗は赤。クリーンな線画、ラベルは日本語
- 同じ見出しを2回描かない。タイトルや無関係な英単語・長い JSON 文字列など、指定外の文字は入れない
- 16:9 横長レイアウト
```
