# 7-1-web-vs-api-routes

**モデル**: pro | **アスペクト比**: 16:9 | **解像度**: 2k

## プロンプト本文

```
同じデータに対する Web ルートと API ルートの違いを、2つの入口の対比で表す立体感のある概念図を作成してください。

【内容】
- 中央下に立体的なデータベース（円柱を重ねた形）を1つ置き「tasks」とラベル。ここから左右2つの入口へデータが伸びる
- 左側の入口（Web ルート）: 立体的なドアの上に「routes/web.php」。ドアの前に人物がブラウザを見ている。ドアには回転式ゲート（CSRF・セッションの関門）。出てくるものは HTML の画面。下に「人・ブラウザ → HTML」とラベル
- 右側の入口（API ルート）: 立体的なドアの上に「routes/api.php」。ドアの前にプログラム（ロボットやスマホアプリのアイコン）。ゲートはなく素通り（ステートレス・CSRFなし）。出てくるものは波括弧 { } の JSON カード。下に「プログラム → JSON」とラベル

【スタイル】
- フラットなインフォグラフィック／ベクター風を基調に、アイソメトリック（斜め上視点）で立体感を出す（ドア・データベースに厚みを持たせる）
- 影はオブジェクト真下の単色フラット影のみ。ぼかし影・グラデーションは使わない
- 背景は #FFFFFF の単色。テクスチャ・しみ・もや・かすれを描かない
- 左（Web）はグレー寄りの青、右（API）はオレンジのアクセントで分ける。クリーンな線画、ラベルは日本語（routes/web.php・routes/api.php・JSON・HTML はそのまま）
- タイトルや無関係な英単語など、指定外の文字は入れない
- 16:9 横長レイアウト
```
