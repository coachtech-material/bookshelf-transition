## 🎯 このセクションで学ぶこと

- `withCount` / `withAvg` / `withSum` で、関連レコードの件数・平均・合計を集計する

このセクションでは、集計のためのメソッドを学び、サンドボックスの Tinker で実際にカテゴリ別の集計とランキングを作ります。

## 導入: 「カテゴリごとのタスク数」をどう数えるか

カテゴリの一覧に「このカテゴリには何件のタスクがあるか」を添えたい、という場面はよくあります。素朴に書くと、カテゴリを 1 つずつ取り出して `$category->tasks->count()` のように数える形になりがちです。しかしこれは、カテゴリが 10 件あればタスクを数えるための問い合わせが 10 回走る、という非効率（N+1 問題）につながります。

数える・平均する・合計する、といった集計は、**データベースに任せる** のが定石です。Laravel には、関連レコードを集計してモデルに添える専用のメソッドがあります。これを使えば、件数つきの一覧を効率よく取得でき、さらに集計結果での並べ替え（ランキング）も簡単に作れます。

### 🧠 先輩エンジニアの思考プロセス

> カテゴリ一覧にタスク数を出すのに、最初はカテゴリごとに `count()` を呼んでいました。カテゴリが増えるほど問い合わせが増え、一覧が目に見えて重くなる。`withCount` を 1 つ足すだけで件数つきの一覧が取れると知ってから、「数えるのは PHP の仕事ではなくデータベースの仕事」と考えを改めました。

---

## withCount で件数を数える

`withCount` は、関連レコードの **件数** を数えて、各モデルに `{関連名}_count` という属性として添えます。`tasks` 関連なら `tasks_count` です。

```php
// 各カテゴリに、タスクの件数を添えて取得する
$categories = Category::withCount('tasks')->get();

$categories->first()->tasks_count; // 例: 3
```

ポイントは、これが **1 回のまとまった問い合わせ** で件数を求めることです。カテゴリごとに数え直すループは発生しません。一覧画面で「各カテゴリの件数」を出すような場面に向きます。

## withAvg と withSum で平均・合計を求める

`withCount` の仲間に、平均を求める `withAvg` と、合計を求める `withSum` があります。これらは「どの列を集計するか」を第 2 引数で指定します。添えられる属性名は `{関連名}_avg_{列名}` / `{関連名}_sum_{列名}` です。

```php
// タスクの見積もり時間（estimated_minutes）の合計・平均を添える
Category::withSum('tasks', 'estimated_minutes')->get(); // tasks_sum_estimated_minutes
Category::withAvg('tasks', 'estimated_minutes')->get(); // tasks_avg_estimated_minutes
```

🔑 件数（`withCount`）は対象が 0 件でも `0` ですが、合計・平均（`withSum` / `withAvg`）は対象が **0 件だと `null`** になります。「タスクが 1 件もないカテゴリの平均」は計算できないため、`0` ではなく `null` で返る、と覚えてください。

## has と whereHas で絞り込む

集計とあわせてよく使うのが、「関連を持っているか」で絞り込む `has` と、「条件を満たす関連を持っているか」で絞り込む `whereHas` です。

```php
// タスクを 1 件以上持つカテゴリだけを取得
Category::has('tasks')->get();

// 「見積もり 60 分以上のタスク」を持つカテゴリだけを取得
Category::whereHas('tasks', function ($query) {
    $query->where('estimated_minutes', '>=', 60);
})->get();
```

`has('tasks')` は「タスクが 1 件でもあるか」を見ます。`whereHas` は、クロージャの中で関連側の条件を指定し、その条件を満たす関連を持つものだけに絞ります。ランキングで「タスクが 0 件のカテゴリは載せない」といった除外に使えます。

## orderByDesc で集計結果を並べ替える＝ランキング

集計した値は、ふつうの列と同じように `orderByDesc`（降順）で並べ替えられます。`withCount` が添える `tasks_count` を降順に並べれば、「タスクが多い順」のランキングになります。

```php
// タスクが多い順（タスクのあるカテゴリだけ・上位 2 件）
Category::withCount('tasks')
    ->has('tasks')
    ->orderByDesc('tasks_count')
    ->take(2)
    ->get();
```

これがランキングの基本形です。集計（`withCount`）で並べ替えの基準を作り、`has` で対象を絞り、`orderByDesc` で順序を付け、`take` で上位だけを取り出します。`take(n)` は先頭から `n` 件だけを取得するメソッドです。

## 🏃 実践: サンドボックスの Tinker で集計する

### 🏃 Step 1: Category と Task のマイグレーションを用意する

`Category` と `Task` のモデルとマイグレーションを作ります。

```bash
# laravel-sandbox ディレクトリで実行
sail artisan make:model Category -m
sail artisan make:model Task -m
```

`categories` テーブルのマイグレーションを開き、内容を次のように **まるごと書き換えます** （`name` 列を加えます）。

```php
// database/migrations/xxxx_xx_xx_xxxxxx_create_categories_table.php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('categories', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('categories');
    }
};
```

`tasks` テーブルのマイグレーションを開き、カテゴリへの外部キーと、集計に使う `estimated_minutes`（見積もり時間・分）を加えた形に **まるごと書き換えます**。

```php
// database/migrations/xxxx_xx_xx_xxxxxx_create_tasks_table.php
<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    public function up(): void
    {
        Schema::create('tasks', function (Blueprint $table) {
            $table->id();
            $table->foreignId('category_id')->constrained()->onDelete('cascade');
            $table->string('title');
            $table->integer('estimated_minutes');
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('tasks');
    }
};
```

### 🏃 Step 2: マイグレーションを実行する

```bash
# laravel-sandbox ディレクトリで実行
sail artisan migrate
```

`categories` と `tasks` が追加されます（`categories` が先に作られるので、`tasks` の外部キーも問題なく張れます）。

### 🏃 Step 3: モデルにリレーションを定義する

`Category` モデル（`app/Models/Category.php`）を次の内容にします。

```php
// app/Models/Category.php
namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\HasMany;

class Category extends Model
{
    use HasFactory;

    protected $fillable = ['name'];

    public function tasks(): HasMany
    {
        return $this->hasMany(Task::class);
    }
}
```

`Task` モデル（`app/Models/Task.php`）を次の内容にします。

```php
// app/Models/Task.php
namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;

class Task extends Model
{
    use HasFactory;

    protected $fillable = ['category_id', 'title', 'estimated_minutes'];

    public function category(): BelongsTo
    {
        return $this->belongsTo(Category::class);
    }
}
```

### 🏃 Step 4: Tinker でデータを用意する

Tinker を起動し、カテゴリ 3 つと、それぞれにタスクを作ります（`hobby` カテゴリはあえて 0 件にします）。Tinker の表示では日本語がバイト列になることがあるため、確認しやすいよう **英語のラベル** を使います。

```bash
# laravel-sandbox ディレクトリで実行
sail artisan tinker
```

```php
$work = \App\Models\Category::create(['name' => 'work']);
$home = \App\Models\Category::create(['name' => 'home']);
$hobby = \App\Models\Category::create(['name' => 'hobby']);

// work カテゴリに 3 件（60 / 30 / 90 分）
$work->tasks()->create(['title' => 'report', 'estimated_minutes' => 60]);
$work->tasks()->create(['title' => 'email', 'estimated_minutes' => 30]);
$work->tasks()->create(['title' => 'meeting', 'estimated_minutes' => 90]);

// home カテゴリに 1 件（45 分）
$home->tasks()->create(['title' => 'shopping', 'estimated_minutes' => 45]);

// hobby カテゴリは 0 件のまま
```

`$work->tasks()->create([...])` は、`hasMany` 関連を通じてタスクを作り、`category_id` を自動でそのカテゴリに設定します。

### 🏃 Step 5: withCount / withSum / withAvg を試す

💡 **Tinker での 2 つの注意**: ① 1 つの文は **1 行で書きます** （メソッドチェーンを複数行に分けて貼ると、Tinker は行ごとに実行し、2 行目以降の `->` で始まる行が構文エラーになります）。② Tinker の表示では、**日本語などのマルチバイト文字が `<E4><BB><95>` のようなバイト列になることがあります** （データ自体は正しく、画面や API では正常に表示されます）。この実験では結果を読みやすくするため、カテゴリ名に英語のラベル（`work` など）を使っています。

まず `withCount` で件数を数えます。`foreach` で 1 行ずつ出力します。

```php
foreach (\App\Models\Category::withCount('tasks')->get() as $c) echo "{$c->name}: {$c->tasks_count}\n";
// work: 3
// home: 1
// hobby: 0
```

hobby カテゴリは 0 件でも `0` が入っています。次に、work カテゴリ（id 1）の合計・平均を見ます。チェーンは **1 行** で書きます（改行して貼ると `->` の行で構文エラーになります）。

```php
$work = \App\Models\Category::withSum('tasks', 'estimated_minutes')->withAvg('tasks', 'estimated_minutes')->find(1);
(int) $work->tasks_sum_estimated_minutes;
// => 180
round((float) $work->tasks_avg_estimated_minutes, 1);
// => 60.0
```

💡 集計の値は、データベースの都合で文字列や小数として返ることがあります（たとえば平均は `60.0000` のような小数の文字列）。そのため、ここでは `(int)` や `round()` で整えています。画面や API で使うときも、この整え方がそのまま役立ちます（API での整形は 7-2 で扱います）。

0 件のカテゴリでは、合計・平均が `null` になることも確認します。

```php
\App\Models\Category::withSum('tasks', 'estimated_minutes')->find(3)->tasks_sum_estimated_minutes;
// => null
```

### 🏃 Step 6: has / whereHas とランキングを試す

`has` で「タスクがあるカテゴリだけ」を取得します。hobby は 0 件なので除外されます。

```php
foreach (\App\Models\Category::has('tasks')->get() as $c) echo "{$c->name}\n";
// work
// home
```

`whereHas` で「60 分以上のタスクを持つカテゴリだけ」を取得します。work は 60 分・90 分のタスクを持つので該当し、home は 45 分だけなので外れます。クロージャは **アロー関数** にして 1 行で書きます。

```php
foreach (\App\Models\Category::whereHas('tasks', fn ($q) => $q->where('estimated_minutes', '>=', 60))->get() as $c) echo "{$c->name}\n";
// work
```

最後に、ランキングを作ります。「タスクが多い順・タスクのあるカテゴリだけ・上位 2 件」です。

```php
foreach (\App\Models\Category::withCount('tasks')->has('tasks')->orderByDesc('tasks_count')->take(2)->get() as $c) echo "{$c->name}: {$c->tasks_count}\n";
// work: 3
// home: 1
```

タスクが多い順に、work（3 件）・home（1 件）が並びました。確認できたら `exit` で Tinker を終了します。

### 🏃 Step 7: 後片付けをする

このセクションの実験も使い捨てです。追加したテーブルとファイルを元に戻します。


## ✨ まとめ

- `withCount` は件数を `{関連名}_count` として添える。ループで数えず、データベースに集計させる
- `withAvg` / `withSum` は平均・合計を `{関連名}_avg_{列名}` / `{関連名}_sum_{列名}` として添える。0 件だと `null` になる
- `has` は関連の有無、`whereHas` は条件を満たす関連の有無で絞り込む
- `withCount` + `orderByDesc` + `take`（必要なら `has` で除外）でランキングを作れる
- 集計値は文字列や小数で返ることがあるため、使うときは `(int)` や `round()` で整える
