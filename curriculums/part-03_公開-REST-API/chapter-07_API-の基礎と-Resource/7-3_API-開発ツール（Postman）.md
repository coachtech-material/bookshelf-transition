# 7-3 API 開発ツール（Postman）

📝 **前提知識**: このセクションは 7-1 REST API とルート設計 と 2-3 ハンズオン: Laravel Sail プロジェクトを立ち上げる の内容を前提としています。

## 🎯 このセクションで学ぶこと

- API テストツールがなぜ必要かを理解する
- Postman をインストールし、GET / POST などのリクエストを送る
- リクエストヘッダー（`Accept: application/json`）とボディ（JSON）を設定する
- コレクションと環境変数でリクエストを整理・再利用する

このセクションでは、Postman をインストールし、2-3 で立ち上げたサンドボックスに用意した API へ実際にリクエストを送って、レスポンスを確認します。

---

## 導入: 概念は分かった。でも、どうやって試す？

7-1・7-2 で、API のルートとレスポンスの形を学びました。では、作った API が本当に意図どおりに動くか、どうやって確かめるのでしょうか。

ブラウザのアドレスバーに URL を打ち込めば、`GET` のリクエストは送れます。しかし、新規作成の `POST`、更新の `PUT`、削除の `DELETE` は、アドレスバーからは送れません。これらを送り、レスポンスの JSON を見やすく確認するには、専用のツールが要ります。ここで使うのが **Postman** です。

### 🧠 先輩エンジニアの思考プロセス

> API を作り始めた頃、ブラウザのアドレスバーに URL を打ち込んで確認していて、`POST` や `DELETE` をどう送ればいいのか分からず手が止まりました。Postman を知ってからは、メソッドを選んでボディを書いて送るだけ。しかもコレクションに保存しておけば、毎回打ち直さずに済むし、チームにそのまま渡せます。テストツールは「あると便利」ではなく、API 開発の必需品だと考えています。

---

## なぜ API テストツールが必要か

画面（HTML）を返す Web アプリは、ブラウザで開けば動作を確認できました。API は JSON を返すので、確認のしかたが変わります。専用ツールが要る理由は 3 つです。

- **ブラウザでは送れないリクエストがある**: アドレスバーから送れるのは `GET` だけです。`POST` / `PUT` / `DELETE` や、リクエストボディ（JSON）を伴う送信はできません。
- **リクエストを保存して繰り返せる**: 開発中は同じリクエストを何度も送ります。ツールに保存しておけば、毎回 URL やボディを打ち直さずに済みます。
- **チームで共有できる**: 保存したリクエストの一式を共有すれば、チーム全員が同じ条件で API を試せます。

## Postman とは

**Postman** は、API にリクエストを送り、レスポンスを確認するためのツールです。API 開発で広く使われており、無料の範囲でも本教材の確認には十分です。

メソッド（GET / POST など）・URL・ヘッダー・ボディを画面で設定して「Send」を押すと、レスポンスの JSON が整形されて表示されます。さらに、リクエストを **コレクション** にまとめて保存したり、URL の共通部分を **環境変数** として使い回したりできます。

💡 Postman のほかにも、VS Code 拡張機能の Thunder Client など、同じ役割のツールがあります。本教材では Postman を使いますが、考え方（メソッド・URL・ヘッダー・ボディを設定して送る）はどのツールでも同じです。

## 🏃 実践: Postman で API を試す

### 🏃 Step 1: Postman をインストールする

Postman の公式サイト（`https://www.postman.com/`）からデスクトップアプリをダウンロードし、インストールします（2026年6月時点では「Download」から各 OS 向けのインストーラーを入手できます）。起動後、アカウントを作成してログインします（Google アカウントなどでもログインできます）。

💡 画面の細部は更新で変わることがあります。最新の手順は公式サイトの案内に従ってください。基本の流れ（リクエストを作り、メソッド・URL を設定して Send する）は変わりません。

### 🏃 Step 2: サンドボックスに確認用のルートを足す

`laravel-sandbox` の `routes/api.php` を開き、ファイルの末尾に確認用のルートを 2 つ追加します。1 つは `GET` で固定の JSON を返すルート、もう 1 つは `POST` で受け取ったボディをそのまま返すルートです。

```php
// routes/api.php （末尾に追加・確認用の使い捨て）
Route::get('/ping', function () {
    return response()->json(['message' => 'pong']);
});

Route::post('/echo', function (Request $request) {
    return response()->json(['received' => $request->all()]);
});
```

📝 `routes/api.php` の先頭には、もともと `use Illuminate\Http\Request;` が書かれています。そのため `POST` のルートで `Request $request` を型宣言できます。7-1 で見たとおり、`routes/api.php` のルートには自動で `/api` が付くので、URL はそれぞれ `/api/ping`・`/api/echo` になります。

### 🏃 Step 3: GET リクエストを送る

Postman で新しいリクエストを作り（`New` → `HTTP`、または `+` タブ）、次のように設定して `Send` を押します。

- メソッド: `GET`
- URL: `http://localhost/api/ping`

レスポンスに、次の JSON とステータス `200 OK` が表示されれば成功です。

```json
{
  "message": "pong"
}
```

ブラウザのアドレスバーでも `GET` は確認できますが、Postman ならステータスコードやヘッダーも一緒に見られます。

### 🏃 Step 4: POST リクエストを送る（JSON ボディ）

次に、ボディ付きの `POST` を送ります。新しいリクエストを作り、次のように設定します。

- メソッド: `POST`
- URL: `http://localhost/api/echo`
- `Headers` タブで `Accept` に `application/json` を設定する
- `Body` タブで `raw` を選び、右の形式で `JSON` を選んで、次の内容を入力する

```json
{
  "title": "買い物",
  "done": false
}
```

`Send` を押すと、送ったボディがそのまま返ってきます。

```json
{
  "received": {
    "title": "買い物",
    "done": false
  }
}
```

🔑 `Accept: application/json` ヘッダーは、「JSON で応えてほしい」とサーバーに伝えるものです。これを付けないと、バリデーション失敗時に API が JSON ではなくリダイレクトを返すことがあります（なぜそうなるかは 8-2 で扱います）。API を Postman で叩くときは、このヘッダーを付けるのを習慣にしてください。

### 🏃 Step 5: コレクションと環境変数でまとめる

開発中は同じリクエストを繰り返し送るので、保存して使い回せるようにします。

- **コレクション**: 関連するリクエストをまとめる入れ物です。`Save` でリクエストをコレクションに保存すると、次回からクリックするだけで送れます。
- **環境変数**: URL の共通部分（`http://localhost` など）を変数にしておく仕組みです。環境（Environment）を 1 つ作り、`base_url` という変数に `http://localhost` を設定します。リクエストの URL では `{{base_url}}/api/ping` のように `{{ }}` で囲んで使います。

環境変数を使うと、後で接続先が変わっても、変数を 1 か所変えるだけですべてのリクエストに反映できます。次の 8-3 では、この `base_url` を使って公開タスク API を叩きます。

### 🏃 Step 6: 後片付けをする

確認が終わったら、Step 2 で `routes/api.php` の末尾に足した `/ping` と `/echo` の 2 つのルートを削除し、ファイルを元の状態に戻します。これらは確認用の使い捨てなので、残しておく必要はありません。

Postman に保存したコレクションと環境（`base_url`）は、次の 8-3 でも使うので残しておいて構いません。

## ✨ まとめ

- API は JSON を返すため、`POST` / `PUT` / `DELETE` も送れる専用ツールで確認する。ブラウザのアドレスバーでは `GET` しか送れない
- Postman は、メソッド・URL・ヘッダー・ボディを設定して `Send` するだけでレスポンスを確認できる
- `Accept: application/json` ヘッダーを付けて「JSON で応えてほしい」と伝える
- コレクションでリクエストを保存し、環境変数（`{{base_url}}`）で接続先を使い回す
