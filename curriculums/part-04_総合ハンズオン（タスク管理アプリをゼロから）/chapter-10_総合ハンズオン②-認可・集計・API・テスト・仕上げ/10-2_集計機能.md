# 10-2 集計機能

📝 **このハンズオンで使う機能**: `withCount` / `loadCount`（5-1 集計クエリとランキング で学習）、`with` / `load` の使い分け（5-2 Eager Loading の使い分けと N+1 で学習）

📝 **前提知識**: このセクションは 5-1 集計クエリとランキング と 5-2 Eager Loading の使い分けと N+1 と 9-5 CRUD（カテゴリ・タスク）と多対多操作 の内容を前提としています。

## 🎯 このセクションで学ぶこと

- よく使われているタグのランキングを `withCount` + `orderByDesc` で実装する
- タスクの詳細画面に、所属カテゴリのタスク数を `loadCount` で表示する
- 一覧は `with` / `withCount`、詳細は `load` / `loadCount` という使い分けで N+1 を避ける

このセクションでは、集計機能を実装します。9-2 で配置したランキング画面に命を吹き込み、詳細画面に集計値を表示します。

---

## 導入: 「よく使われているタグ」を効率よく数える

このアプリには、タスクとタグの多対多があります。これを集計すれば、「どのタグが多くのタスクに使われているか」というランキングや、「このカテゴリには何件のタスクがあるか」という指標が作れます。

素朴に書くと、タグごとに `$tag->tasks->count()` で数えたくなりますが、それは 5-1 で見た N+1 問題につながります。集計はデータベースに任せるのが定石です。一覧（ランキング）では `withCount`、すでに取得した 1 件（詳細画面）では `loadCount` を使い分けます。9-5 でカテゴリ一覧に使った `withCount('tasks')`（カテゴリ別タスク数）も、同じ集計の仲間です。

### 🧠 先輩エンジニアの思考プロセス

> ランキングのような「集計して並べる」処理は、PHP のループで数えると、データが増えた瞬間に重くなります。`withCount` で数えて `orderByDesc` で並べる、という 1 本のクエリにまとめると、件数が増えても速度が落ちません。「数えるのも並べるのもデータベースの仕事」と割り切るのが、集計を速く保つコツです。

## 🏃 実践: タグランキングとカテゴリ集計を実装する

### 🏃 Step 1: RankingController を実装する

ランキングのルート（`ranking.index`）は 9-2 で定義済みです。対応するコントローラを作ります。

```bash
# task-app ディレクトリで実行
sail artisan make:controller RankingController
```

`app/Http/Controllers/RankingController.php` を **まるごと書き換えます**。タスクに使われている数が多い順に、上位 10 件のタグを取得します。

```php
<?php
// app/Http/Controllers/RankingController.php

namespace App\Http\Controllers;

use App\Models\Tag;
use Illuminate\View\View;

class RankingController extends Controller
{
    public function index(): View
    {
        $rankedTags = Tag::withCount('tasks')
            ->has('tasks')
            ->orderByDesc('tasks_count')
            ->take(10)
            ->get();

        return view('ranking.index', compact('rankedTags'));
    }
}
```

このクエリを、5-1 で学んだ部品で読み解きます。

- **`withCount('tasks')`**: 各タグに、そのタグが付いたタスクの **件数** を添えます。`Tag::tasks()` は `task_tag` を介した多対多なので、これは「多対多の相手を数える」集計です。`withCount` は既定で `{関連名}_count` という属性名を使うので、`tasks` なら `tasks_count` になります。
- **`has('tasks')`**: タスクが 1 件も付いていないタグを、ランキングから除外します。
- **`orderByDesc('tasks_count')`** + **`take(10)`**: 集計した件数の降順に並べ、上位 10 件を取り出します。

🔑 これがランキングの基本形です。集計（`withCount`）で並べ替えの基準を作り、`has` で対象を絞り、`orderByDesc` で順序を付け、`take` で上位だけを取り出す。5-1 では `hasMany`（カテゴリのタスク数）を数えましたが、ここでは **多対多** （タグに付いたタスク数）を数えています。`withCount` は、1 対多でも多対多でも同じ書き方で使えます。9-2 で配置した `ranking/index.blade.php` は `$tag->tasks_count` を表示します。

ブラウザでナビの「ランキング」を開きます。タグが、使われているタスク数の多い順（9-3 のシーダーなら 重要・今週中・緊急・アイデア の順）に並べば成功です。

### 🏃 Step 2: 詳細画面にカテゴリのタスク数を表示する

タスクの詳細画面に、「このタスクが属するカテゴリには、全部で何件のタスクがあるか」を表示します。一覧（ランキング）では `withCount` を使いましたが、詳細画面は **すでに取得した 1 件** に後付けするので、5-2 で学んだとおり `loadCount` を使います。

`app/Http/Controllers/TaskController.php` の `show` メソッドに、`loadCount` の 1 行を加えます（`show` 全体は次のようになります）。読み込み済みのカテゴリに、タスク件数を後付けします。

```php
// app/Http/Controllers/TaskController.php （show メソッドに loadCount を追加）
    public function show(Task $task): View
    {
        $task->load(['category', 'tags', 'user']);
        $task->category->loadCount('tasks');

        return view('tasks.show', compact('task'));
    }
```

🔑 `withCount`（クエリで取得する前）と `loadCount`（取得済みのモデルに後付け）は、どちらも同じ集計を求めます。`$task->load('category')` で読み込んだカテゴリは取得済みのモデルなので、そこに件数を足すには `loadCount` を使います。`$task->category->loadCount('tasks')` で、そのカテゴリに `tasks_count`（カテゴリに属するタスク数）が添えられます。

次に、`resources/views/tasks/show.blade.php` のカテゴリ表示に、件数を加えます。カテゴリを表示している 1 行を探し、次の形に **置き換えます**。

```blade
{{-- resources/views/tasks/show.blade.php のカテゴリ行（置き換え後） --}}
            <div class="flex"><dt class="w-24 text-gray-500">カテゴリ</dt><dd>{{ $task->category->name }}（全 {{ $task->category->tasks_count }} 件）</dd></div>
```

### 🏃 Step 3: 動作を確認する

ブラウザで確認します。

1. ナビの「ランキング」を開く。タグが、付いているタスク数の多い順に並ぶ。タスクが 0 件のタグは出ない（`has` で除外）。
2. タスクの詳細画面を開く。カテゴリ名の横に「（全 ○ 件）」と、同じカテゴリのタスク数が表示される。
3. タスクを新しく登録して同じカテゴリを選ぶと、そのカテゴリのタスクの詳細画面で件数が 1 増える。タグを付け替えると、ランキングの順位も変わる。

💡 9-5 で実装したカテゴリ一覧の「タスク数」も、`withCount('tasks')` による同じ集計です。一覧でまとめて数える `withCount`、取得済みの 1 件に後付けする `loadCount`、という使い分けが、このアプリの集計の土台になっています。

## ✨ まとめ

- ランキングは `withCount('関連')` + `has` + `orderByDesc` + `take` で実装する。集計と並べ替えをデータベースに任せ、N+1 を避ける
- `withCount` は 1 対多でも多対多でも同じ書き方で使え、既定で `{関連名}_count` という属性を添える（必要なら `as` で別名を付けられる）
- 一覧（コレクション）には `withCount`、詳細（取得済みの 1 件）には `loadCount`。9-5 のカテゴリ別タスク数（`withCount('tasks')`）も同じ集計の仲間
- 関連の表示には `with` / `load` を添えて N+1 を避ける
