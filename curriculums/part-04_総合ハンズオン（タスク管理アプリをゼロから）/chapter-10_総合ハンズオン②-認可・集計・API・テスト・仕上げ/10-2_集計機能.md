# 10-2 集計機能

📝 **このハンズオンで使う機能**: `withCount` / `loadCount`（5-1 集計クエリとランキング で学習）、`with` / `load` の使い分け（5-2 Eager Loading の使い分けと N+1 で学習）

📝 **前提知識**: このセクションは 5-1 集計クエリとランキング と 5-2 Eager Loading の使い分けと N+1 と 9-5 CRUD（カテゴリ・タスク）と多対多操作 の内容を前提としています。

## 🎯 このセクションで学ぶこと

- お気に入りの多い順のランキングを `withCount` + `orderByDesc` で実装する
- タスクのお気に入り数を、詳細画面に `loadCount` で表示する
- 一覧は `with` / `withCount`、詳細は `load` / `loadCount` という使い分けで N+1 を避ける

このセクションでは、集計機能を実装します。9-2 で配置したランキング画面に命を吹き込み、詳細画面にお気に入り数を表示します。

---

## 導入: 「人気のタスク」を効率よく数える

このアプリには、お気に入りという多対多があります。これを集計すれば、「お気に入りの多い順」のランキングや、「このタスクは何人にお気に入りされているか」という人気の指標が作れます。

素朴に書くと、タスクごとに `$task->favoritedByUsers->count()` で数えたくなりますが、それは 5-1 で見た N+1 問題につながります。集計はデータベースに任せるのが定石です。一覧（ランキング）では `withCount`、すでに取得した 1 件（詳細画面）では `loadCount` を使い分けます。9-5 でカテゴリ一覧に使った `withCount('tasks')`（カテゴリ別タスク数）も、同じ集計の仲間です。

### 🧠 先輩エンジニアの思考プロセス

> ランキングのような「集計して並べる」処理は、PHP のループで数えると、データが増えた瞬間に重くなります。`withCount` で数えて `orderByDesc` で並べる、という 1 本のクエリにまとめると、件数が増えても速度が落ちません。「数えるのも並べるのもデータベースの仕事」と割り切るのが、集計を速く保つコツです。

## 🏃 実践: ランキングとお気に入り数を実装する

### 🏃 Step 1: RankingController を実装する

ランキングのルート（`ranking.index`）は 9-2 で定義済みです。対応するコントローラを作ります。

```bash
# task-app ディレクトリで実行
sail artisan make:controller RankingController
```

`app/Http/Controllers/RankingController.php` を **まるごと書き換えます**。お気に入りの多い順に、上位 10 件を取得します。

```php
// app/Http/Controllers/RankingController.php
<?php

namespace App\Http\Controllers;

use App\Models\Task;
use Illuminate\View\View;

class RankingController extends Controller
{
    public function index(): View
    {
        $rankedTasks = Task::withCount(['favoritedByUsers as favorites_count'])
            ->has('favoritedByUsers')
            ->with('category')
            ->orderByDesc('favorites_count')
            ->take(10)
            ->get();

        return view('ranking.index', compact('rankedTasks'));
    }
}
```

このクエリを、5-1 で学んだ部品で読み解きます。

- **`withCount(['favoritedByUsers as favorites_count'])`**: 各タスクに、お気に入りしたユーザーの **件数** を添えます。`withCount` は既定で `{関連名}_count` という属性名を使うので、`favoritedByUsers` だと `favorited_by_users_count` という長い名前になります。`as favorites_count` で別名を付け、`favorites_count` という短い名前で受け取れるようにしています。
- **`has('favoritedByUsers')`**: お気に入りが 1 件もないタスクを、ランキングから除外します。
- **`with('category')`**: ランキング画面でカテゴリ名を表示するため、関連を Eager Loading します（N+1 回避）。
- **`orderByDesc('favorites_count')`** + **`take(10)`**: 集計した件数の降順に並べ、上位 10 件を取り出します。

🔑 これがランキングの基本形です。集計（`withCount`）で並べ替えの基準を作り、`has` で対象を絞り、`orderByDesc` で順序を付け、`take` で上位だけを取り出す。9-2 で配置した `ranking/index.blade.php` は `$task->favorites_count` を表示するので、別名を `favorites_count` にそろえています。

ブラウザでナビの「ランキング」を開きます。お気に入りされたタスクが、お気に入り数（♥ の数）の多い順に並べば成功です。

### 🏃 Step 2: 詳細画面にお気に入り数を表示する

タスクの詳細画面に、「何人にお気に入りされているか」を表示します。一覧（ランキング）では `withCount` を使いましたが、詳細画面は **すでに取得した 1 件** に後付けするので、5-2 で学んだとおり `loadCount` を使います。

`app/Http/Controllers/TaskController.php` の `show` メソッドに、`loadCount` の 1 行を加えます（`show` 全体は次のようになります）。

```php
// app/Http/Controllers/TaskController.php （show メソッドに loadCount を追加）
    public function show(Task $task): View
    {
        $task->load(['category', 'tags', 'user']);
        $task->loadCount(['favoritedByUsers as favorites_count']);

        return view('tasks.show', compact('task'));
    }
```

🔑 `withCount`（クエリで取得する前）と `loadCount`（取得済みのモデルに後付け）は、どちらも同じ集計を求めます。ルートモデルバインディングで受け取った `$task` はすでに取得済みなので、`load` 系を使うのが自然です。別名（`as favorites_count`）も同じにそろえることで、ビュー側は一覧でも詳細でも `$task->favorites_count` で受け取れます。

次に、`resources/views/tasks/show.blade.php` のお気に入りボタンの部分に、件数の表示を加えます。9-2 で配置したお気に入りトグルのブロック（`<form>` から `</form>` まで）を、件数表示を加えた次の形に **置き換えます**。

```blade
{{-- resources/views/tasks/show.blade.php のお気に入りトグル（置き換え後） --}}
            <div class="text-right">
                <form action="{{ route('favorites.toggle', $task) }}" method="POST">
                    @csrf
                    @if (Auth::user()->favoriteTasks->contains($task->id))
                        <button type="submit" class="text-red-500 hover:text-red-700" title="お気に入りから外す">♥ お気に入り中</button>
                    @else
                        <button type="submit" class="text-gray-400 hover:text-red-500" title="お気に入りに追加">♡ お気に入り</button>
                    @endif
                </form>
                <p class="mt-1 text-xs text-gray-500">お気に入り {{ $task->favorites_count }} 人</p>
            </div>
```

### 🏃 Step 3: 動作を確認する

ブラウザで確認します。

1. ナビの「ランキング」を開く。お気に入りされたタスクが、♥ の数の多い順に並ぶ。お気に入りが 0 件のタスクは出ない（`has` で除外）。
2. タスクの詳細画面を開く。お気に入りボタンの下に「お気に入り ○ 人」と表示される。
3. お気に入りボタンを押して登録・解除すると、詳細画面を再読み込みしたときに件数が増減する。ランキングの順位も、お気に入りの増減に応じて変わる。

💡 9-5 で実装したカテゴリ一覧の「タスク数」も、`withCount('tasks')` による同じ集計です。一覧でまとめて数える `withCount`、取得済みの 1 件に後付けする `loadCount`、という使い分けが、このアプリの集計の土台になっています。

## ✨ まとめ

- ランキングは `withCount(['関連 as 別名'])` + `has` + `orderByDesc` + `take` で実装する。集計と並べ替えをデータベースに任せ、N+1 を避ける
- `withCount` の既定の属性名（`{関連名}_count`）は、`as` で短い別名に付け替えられる
- 一覧（コレクション）には `withCount`、詳細（取得済みの 1 件）には `loadCount`。9-5 のカテゴリ別タスク数（`withCount('tasks')`）も同じ集計の仲間
- 関連の表示には `with` / `load` を添えて N+1 を避ける
