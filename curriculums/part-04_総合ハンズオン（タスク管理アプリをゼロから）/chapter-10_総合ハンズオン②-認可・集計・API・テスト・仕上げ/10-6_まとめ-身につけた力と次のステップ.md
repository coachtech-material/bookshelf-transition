## この教材で身につけた力

この教材を通して、次の技術を、概念の理解と手を動かす実践の両面から身につけました。

- **Laravel 10 + Sail の開発環境** （Part 1）: 手書きの Docker Compose から Sail へ移り、`sail` コマンドでコンテナを操作できるようになりました。Laravel 8 から 10 への変更点（無名マイグレーションクラス・`lang/` の位置・`$casts` プロパティなど）も押さえ、Laravel のクラスを読み解く PHP OOP の基礎も固めました。
- **多対多リレーションの実践** （Part 2）: ピボットテーブルの 2 パターン（純粋なピボットと一意制約を持つピボット）を設計し、`belongsToMany` と `attach` / `sync` / `toggle` で操作できるようになりました。Part 4 では、タグ付けを `sync`、お気に入りを `toggle` で実装しました。
- **Policy による認可** （Part 2）: 認証（誰か）と認可（何をしてよいか）を区別し、所有者だけが編集・削除できる制御を、`authorize()` と `@can` で実装できるようになりました。
- **集計とパフォーマンス** （Part 2）: `withCount` で件数を集計し、`orderByDesc` でランキングを作り、`with` / `load` の使い分けで N+1 を避けられるようになりました。
- **自動テスト** （Part 2・Part 4）: `RefreshDatabase`・Factory・`actingAs` を土台に、CRUD・認可・バリデーション・公開 API のテストを書き、カバレッジ 60% 以上を達成できるようになりました。
- **公開 REST API** （Part 3・Part 4）: `routes/api.php` と `apiResource`・バージョニング、API Resource によるレスポンス整形、検索・絞り込み・ページネーション、HTTP ステータスと JSON 例外まで、公開 API を設計から実装まで扱えるようになりました。
- **あいまいな要件を設計に落とす力** （Part 4）: 要件の行間（多重度・所有者・削除の連鎖）を読み解き、ER 図・テーブル・画面/API・バリデーションへと設計できるようになりました。

1-1 で「これから学ぶこと」として挙げた技術が、すべて「できること」に変わりました。

## 新しい模擬案件にどう臨むか

新しい模擬案件に取り組むときも、Part 4 で通した流れがそのまま指針になります。要件が変わっても、進め方は変わりません。

1. **要件を読み解く**: 何を作るかを把握し、書かれていない部分（多重度・所有者・削除時の挙動など）を、設計上の判断として決める（9-1 で行ったこと）。
2. **データ構造を設計する**: ER 図とテーブルを起こし、1 対多・多対多・外部キー・カスケードを決める。多対多があれば、純粋なピボットか一意制約付きかを選ぶ。
3. **環境を整える**: Sail で Laravel を立ち上げ、日本語ロケールと画面の土台を用意する。
4. **積み上げる**: マイグレーションとモデル → 認証 → CRUD → 認可 → 集計 → 公開 API、と機能を 1 つずつ実装する。それぞれの機能に、この教材で学んだ部品（`sync`・`toggle`・Policy・`withCount`・API Resource・JSON 例外）を当てはめる。
5. **テストで固める**: 成功系だけでなく、403・422 といった失敗系のテストを書き、カバレッジを確認する。
6. **仕上げる**: Pint で整形し、README を整え、提出前チェックリストで「ゼロから再現できる」ことを確かめる。

🔑 大切なのは、個々の構文を暗記することではなく、「この要件には、どの技術のどの部品を当てればよいか」を判断できることです。この教材は、その判断ができるよう、各技術を Why → What → How の順で理解し、1 つのアプリに統合する形で学んできました。要件を前にして手が止まったら、この教材の該当セクションに戻り、概念から確かめ直してください。

## 次のステップ

ここで身につけた技術は、Laravel でアプリを作るための確かな土台です。さらに先に進むなら、たとえば次のような方向があります。

- **API 認証**: 今回の公開 API は認証なしでしたが、Laravel Sanctum によるトークン認証を学ぶと、「ログインしたユーザーだけが使える API」を作れます。
- **より進んだテスト**: ブラウザ操作を自動化する Laravel Dusk や、テストを起点に設計する考え方に触れると、テストの幅が広がります。
- **公式ドキュメントを辞書として使う**: 本教材で扱った各機能は、Laravel 公式ドキュメント（10.x）にさらに詳しい使い方が載っています。必要になったときに引ける「辞書」として活用してください。
